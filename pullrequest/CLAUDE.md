# :pullrequest Module

Owns the PR creation dialog, list, detail view, and merge flow for Bitbucket Server.

## Key classes

| Class | Purpose |
|---|---|
| `ui/PrDashboardPanel` | PR list + detail composite. Hosts "+ Create PR" toolbar button. Subscribes to `PullRequestCreated` for auto-refresh. |
| `ui/PrListPanel` | Simple PR list view. |
| `ui/PrDetailPanel` | Single-PR detail with merge/decline actions. |
| `ui/CreatePrDialog` | PR creation modal. Takes a `CreatePrContext` (ticket keys, contexts, transitions, reviewers). Integrates `TicketChipInput`, `ExtendableTextField` with AI-title icon, markdown preview via `MarkdownToHtml`. Multi-ticket `doValidate()`. The dialog opens with an empty description; the user clicks Generate to invoke the AI (the legacy `autoGenerateDescription` flag was dropped in the 2026-04-27 PrBar redesign — Build-tab Create PR no longer exists). |
| `ui/TicketChipInput` | Multi-ticket chip widget (max 5, right-click set-primary, live Jira validation via `getTicketContext`). Implements `Disposable`. |
| `service/PrDescriptionGenerator` | 3-tier generator (AI+diff → AI-no-diff → commit-msg fallback). Accepts `List<TicketContext>`. |
| `service/MarkdownToHtml` | Simple markdown→HTML converter for the dialog Preview pane. |
| `action/CreatePrPrefetch` | Shared prefetch: resolves current branch, remote branches, ticket keys (branch regex + activeTicketId), ticket contexts (parallel), transitions, default title, default reviewers. Returns `PrefetchResult.Success(CreatePrContext)` or `PrefetchResult.Failure(message)`. |
| `action/CreatePrLauncherImpl` | Implementation of `CreatePrLauncher` EP from `:core`. Entry point for PR tab "+ Create PR" button (the only entry point — Build tab's was removed 2026-04-27). |
| `service/PrActionService` / `service/PrDetailService` / `service/PrListService` | Existing list/detail/action services. |

## Architecture

- Single entry point to PR creation: PR tab's "+ Create PR" button (via `CreatePrLauncher.getInstance()?.launch(project, scope)`). Build tab's entry was removed in the 2026-04-27 PrBar redesign — PrBar is now a passive mirror of the PR tab's selection.
- Goes through `CreatePrLauncherImpl` → `CreatePrPrefetch.run(project)` → `CreatePrDialog`.
- Multi-ticket context sent to LLM via `PrDescriptionPromptBuilder` (tiered budget: primary 8000 chars / 10 comments × 400; additional 3000 chars / 3 comments × 300).
- AI title regeneration via `TextGenerationService.generatePrTitle()` gated by `PluginSettings.enableAiTitleGeneration`.

## Threading

- Prefetch + LLM on `Dispatchers.IO` inside `scope.launch`
- Git reads use `readAction { }` (suspending — see `:core` "Service & threading conventions")
- UI mutations via `invokeLater` / `Dispatchers.EDT`
- LLM calls wrapped with `withTimeoutOrNull(120_000)`
- Both `descriptionGenJob` and `titleGenJob` cancelled in `CreatePrDialog.dispose()`
- `PrListService` and `PrDescriptionGenerator` use platform-injected `cs: CoroutineScope`

## Bitbucket Server REST API v1.0

Auth: `Authorization: Bearer <HTTP-access-token>`
Base: `https://{host}/rest/api/1.0/`

Key endpoints:
- `GET /rest/api/1.0/users` — test connection
- `GET /rest/api/1.0/projects/{proj}/repos/{repo}/pull-requests` — list PRs
  - **Note:** requires both `role.1` and `username.1` parameters for author filtering
- `GET /rest/api/1.0/projects/{proj}/repos/{repo}/pull-requests/{id}` — PR detail
- `PUT /rest/api/1.0/projects/{proj}/repos/{repo}/pull-requests/{id}` — update PR
  - **PUT replaces entire PR** — always fetch current state first to preserve title/reviewers
- `GET /rest/api/1.0/projects/{proj}/repos/{repo}/pull-requests/{id}/merge` — merge preconditions (canMerge + vetoes)
- `POST /rest/api/1.0/projects/{proj}/repos/{repo}/pull-requests/{id}/merge` — merge PR
  - Merge strategies: `strategyId` in POST body (e.g., `no-ff`, `squash`, `rebase-no-ff`)
- `POST /rest/api/1.0/projects/{proj}/repos/{repo}/pull-requests/{id}/approve` — approve PR
- `POST /rest/api/1.0/projects/{proj}/repos/{repo}/pull-requests` — create PR

Uses `BitbucketBranchClient` from `:core` — `getBranches`, `getUsers`, `createPullRequest`, `getPullRequestsForBranch`. Auth: `Authorization: Bearer <PAT>` via `CredentialStore`.

## UI

- `PrDashboardPanel` — split view: PR list + detail panel. Hosts "+ Create PR" toolbar button. Subscribes to `PullRequestCreated` for auto-refresh.
- `PrListPanel` — filterable PR list with status indicators
- `PrDetailPanel` — PR metadata, reviewers, merge status, action buttons
- **UI Overhaul:** Outline-style status badges (OPEN/MERGED/DECLINED), sharp card corners, monospace PR IDs, left border accent colored by PR status.
