1. Module Structure & Dependencies

The plugin is a modular monolith: 11 Gradle submodules with a strict dependency rule. Feature modules depend only on :core, never on each other. The one exception is :automation → :bamboo (for BambooApiClient).

2. Cross-Module Communication

All cross-module communication uses the EventBus — a Kotlin MutableSharedFlow with 64-element buffer. Modules emit WorkflowEvent subtypes; other modules subscribe and react.

5. Data Flow: Health Check → Commit

Pre-commit gates: Maven compile, tests, copyright headers, SonarQube gate, and CVE scan all run in parallel before allowing a commit.

sequenceDiagram participant Dev as Developer participant VCS as VcsCheckinHandler participant HC as HealthCheckService participant Maven as Maven Runner participant Copy as Copyright Enforcer participant Sonar as SonarQube participant CVE as CVE Checker Dev->>VCS: Click "Commit" VCS->>HC: runHealthChecks() HC->>HC: emit(HealthCheckStarted) par Parallel checks HC->>Maven: Incremental compile Maven-->>HC: Result and HC->>Maven: Run tests Maven-->>HC: Result and HC->>Copy: Check headers Copy-->>HC: Result and HC->>Sonar: Quality gate Sonar-->>HC: Result and HC->>CVE: Scan pom.xml CVE-->>HC: Result end alt All pass HC->>HC: emit(HealthCheckFinished, passed=true) HC-->>VCS: Allow commit else Any fails HC->>HC: emit(HealthCheckFinished, passed=false) HC-->>VCS: Block commit end

13. Threading Model

IntelliJ enforces strict threading: never block the EDT, never write off the EDT. The plugin uses three execution contexts.

13b. SmartPoller

SmartPoller (in :core) handles all background polling with activity-aware optimizations.

FeatureDetails
Exponential backoff1.5x (build polling) or 2x (queue polling) on consecutive unchanged responses
Jitter±10% random jitter to prevent thundering herd
Visibility gating4x interval when tab is hidden; immediate poll + reset on visibility
Activity resetReturns to base interval on user interaction
Power-saveStops polling when IDE is in power-save mode

21. Agent Architecture (Phase 3)

Overview

The :agent module adds an AI coding assistant powered by Sourcegraph Enterprise's LLM API. It features a ReAct loop with 52 tools, LLM-controlled worker delegation, interactive planning, cross-session memory, and user-extensible skills.

Worker Delegation

The delegate_task tool spawns a WorkerSession with fresh context, a scoped tool subset, and its own budget. Workers cannot delegate further (single-depth). Results are summarized and returned to the parent session.
