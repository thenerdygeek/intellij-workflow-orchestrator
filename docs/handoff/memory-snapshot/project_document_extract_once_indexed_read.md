**SHIPPED 2026-05-27 (committed, NOT pushed) on `feature/cross-ide-delegation`**, commits `db8852780..b0a827918` (16 commits + spec + plan).

**Problem solved:** `read_document` re-extracted the ENTIRE PDF on every chunk (no cache) + ran Tabula lattice on every page → 20-30s/chunk, frequent 30s-timeout retries on big PDFs.

**Architecture (extract-once → persisted artifact + index → cheap slices):**
- `:core` — `DocumentArtifactService` interface + models (`DocumentCursor` {Offset/Page/Section}, `DocumentSlice`, `DocumentIndex` page→offset/heading→offset, `DocumentArtifact(Meta)`, `ExtractionStatus`).
- `:document` — `DocumentArtifactStore` (pure): SHA-256 hash, `extractAndPersist` (full uncapped markdown + index, atomic writes, `meta.json` written LAST = commit sentinel), `slice` (cursor→offset), `writeFailure`/`loadFailureIfFresh` (TTL negative cache). `MarkdownAssembler.assembleIndexed`. `TikaDocumentExtractor.extractBlocks` (refactor: `doExtractToBlocks` + assemble). Lazy-lattice gate in `PdfTableExtractor`.
- `:agent` — `SessionDocumentArtifactService` (`@Service`-style, injected `cs`, single-flight `ConcurrentHashMap<hash,Deferred>`, background job decoupled from read timeout). `DocumentTool` rewritten to delegate (+ `page`/`section` params). `LoopDetector` gained `exemptTools` (read_document exempt — "in progress" retries don't trip doom-loop).
- Cache is **session-scoped** (`sessions/{id}/document-cache/{sha256}/`), content-hash keyed, cleaned on session reset (user chose this over cross-session for privacy). Setting: `documentExtractionJobTimeoutMs` (default 300_000).

**Non-obvious build/test gotchas (verify before editing):**
- `:document` has **NO kotlinx-serialization plugin and NO MockK** — it's a pure module. Use `:core` types' generated `.serializer()` (works, generated in :core); for :document-local serializable use manual JSON tree. Don't add MockK to :document.
- **Document tests must use `runBlocking`, NOT `runTest`** — extractor uses `withTimeoutOrNull` on `Dispatchers.IO` (wall-clock); runTest's virtual time hangs.
- `:agent` consumes `:document` test fixtures (`LargePdfFixtureFactory`, `EncryptedPdfFixtureFactory`) via a `testOutput` configuration added to `document/build.gradle.kts` + `testImplementation(project(":document","testOutput"))` in agent.
- Lazy-lattice gated on Tabula `page.rulings?.isNotEmpty()`; **bypassed when `enableStreamMode=true`** (borderless tables have no rulings). Parity net = existing `PdfTableExtractorTest` (9 tests) unchanged.
- Fixed inner-timeout bug: `extractAndPersist` now takes `extractTimeoutMs` (passed `jobBudgetMs`) so the extractor's default 30s doesn't cap the 5-min job (`6fb496b6d`).
- Running `:core:test` + `:document:test` in ONE gradle invocation fails with a shared-test-sandbox task-validation error — run modules separately.

**⚠ REGRESSION FIXED 2026-05-27 (`ce746e8f2`) — background extraction lost coroutine context:** moving extraction onto `cs.async(Dispatchers.IO)` meant `SessionDownloadDir.current()` (a `CoroutineContext` element installed only around the tool invocation) returned null inside the job → embedded images fell back to `java.io.tmpdir`, which `view_image` (confined to `{sessionDir}/downloads/`) rejects. Fix: capture `coroutineContext[SessionDownloadDir]` in `read()` and launch `cs.async(Dispatchers.IO + sessionCtx)` (element only, NOT the parent Job, so the job still parents to `cs`). **General lesson: `CoroutineScope.async/launch` uses the SCOPE's context, not the caller's — any per-invocation context element (SessionDownloadDir, SessionAttachmentAccess) must be explicitly re-propagated when work is offloaded to a service scope.** Image extraction (PDF XObjects/attachments, DOCX/XLSX/PPTX via POI) emits `[image: path]` markers for `view_image`; HTML/RTF/ODT/EPUB `<img>` = `EmbeddedFileRef(path=null)` reference-only; `ExtractOptions.includeEmbedded` is unused (extraction gated only by imageService being wired). Pinned by `extraction job inherits the SessionDownloadDir context` test.
