EDIT edit_file
Description
Parameters
JSON Schema
Example
Description
Perform an exact string replacement in a file. The old_string must match exactly once in the file. After editing, the file is automatically checked for syntax errors.
Parameters
path string REQUIRED Absolute or project-relative file path
old_string string REQUIRED The exact text to find. Must be unique in the file.
new_string string REQUIRED The replacement text.
JSON Schema (sent to LLM)
{
  "type": "function",
  "function": {
    "name": "edit_file",
    "description": "Perform an exact string replacement...",
    "parameters": {
      "type": "object",
      "properties": {
        "path": { "type": "string" },
        "old_string": { "type": "string" },
        "new_string": { "type": "string" }
      },
      "required": ["path", "old_string", "new_string"]
    }
  }
}
Example Request → Response
LLM sends:
{
  "name": "edit_file",
  "arguments": {
    "path": "src/main/kotlin/UserService.kt",
    "old_string": "return repo.findById(id)",
    "new_string": "return repo?.findById(id)"
  }
}
Agent returns:
// Tool result (sent back to LLM as context)
"Replaced 25 chars with 26 chars in UserService.kt"

// If syntax errors detected (warning):
"Replaced 25 chars... WARNING: 1 syntax error(s):\n Line 23:5: Unexpected token"
