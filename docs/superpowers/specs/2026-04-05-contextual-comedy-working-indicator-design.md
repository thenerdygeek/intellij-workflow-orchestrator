# Contextual Comedy Working Indicator — Design Spec

## Goal
Replace the static "Editing file.kt..." working phrases with humorous, context-aware one-liners generated by Haiku every ~30 seconds while the agent is working.

## Architecture

```
AgentController (30s coroutine timer, starts on executeTask, cancels on completion)
  → HaikuPhraseGenerator.generate(task, recentTools)
    → Haiku API call (async, fire-and-forget)
    → on success: dashboard.setSmartWorkingPhrase(joke)
    → on failure: do nothing (current phrase stays)
```

## How It Works

1. **On task start**, the UI picks a random joke from the existing `WORKING_PHRASES` bank (instant)
2. **Every ~30 seconds**, a background coroutine fires an async Haiku call with the user's task + last 2-3 tool calls
3. Haiku returns a context-aware one-liner. It replaces the current phrase via the existing typewriter animation
4. If Haiku fails or is slow, the current joke stays — user never sees a blank
5. When the agent stops, the timer cancels and phrase clears

## Haiku Prompt (~200 input tokens)

```
You write short, humorous loading messages for an AI coding assistant.
The user asked: "{task}"
Recent activity: {last 2-3 tool names + file paths}

Write ONE short funny loading message (under 60 chars).
Be witty about what's actually happening. No emoji. End with "..."
```

## New Class: HaikuPhraseGenerator

- Lives in `agent/src/main/kotlin/.../observability/HaikuPhraseGenerator.kt`
- Takes `LlmBrainFactory` to create a Haiku-tier brain
- Single `suspend fun generate(task: String, recentTools: List<Pair<String, String>>): String?`
- Returns null on failure (caller ignores)
- Truncates output to 80 chars

## Changes to AgentController

- Add `recentToolCalls: MutableList<Pair<String, String>>` (tool name + file path, max 3, FIFO)
- Populated from `onToolCall` callback (already wired)
- Add a `phraseTimerJob: Job?` that runs every 30s inside `executeTask`
- Timer calls `HaikuPhraseGenerator.generate()` and pushes result to dashboard
- Timer cancels in `onComplete` and `dispose()`
- Gated by `AgentSettings.smartWorkingIndicator` (existing setting, default false)
- When setting is off: existing `buildWorkingPhrase()` static behavior unchanged

## What Doesn't Change

- React UI (WorkingIndicator, chatStore.smartWorkingPhrase, typewriter animation)
- JCEF bridge (setSmartWorkingPhrase)
- Fallback joke bank (WORKING_PHRASES)
- AgentDashboardPanel.setSmartWorkingPhrase()

## Cost

~220 tokens per call. 10-min session = ~20 calls = ~4,400 tokens. Negligible at Haiku pricing.

## Edge Cases

- Task completes in <30s: random bank joke only
- Haiku returns >80 chars: truncate
- Setting off: timer never starts, static phrases remain
- No Haiku model available: generator returns null, fallback stays
